## 主張
余計なファイルをリモートリポジトリにpushしてしまうことがよくある。
適切なファイルを最初から`.gitignore`に過不足なく記載できていることは稀なので、この方法を使うとよい。
## 展開
vaultをgit管理する際に、シークレットが書かれたファイルを間違えてリモートにpushしてしまったので、どうにか管理対象外にできないか調べた。
業務上の開発でも、ローカル管理されるべきファイルをpushしてしまうのはよくあることなので、記憶にしっかり入れようと思った。
## 具体例
```
git rm --cached ファイル名
```
