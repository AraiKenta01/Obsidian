## 主張

`git rm --cached <ファイル名>` を使うと、すでにgitが追跡しているファイルをインデックスから除外し、以降の追跡対象から外せる。ワーキングディレクトリのファイル自体は削除されない。

## 展開

`.gitignore` に記載し忘れたまま `git add` してしまうと、そのファイルはgitの追跡対象になる。この状態で `.gitignore` に追記しても追跡は止まらない。`git rm --cached` でインデックスから除去することで初めて `.gitignore` が有効になる。シークレットキーや環境変数ファイルを誤ってリモートにpushしてしまった場合の対処としても使う。

## 具体例

```bash
# 追跡を止めたいファイルをインデックスから除去
git rm --cached <ファイル名>

# ディレクトリごと除去する場合
git rm --cached -r <ディレクトリ名>
```

除去後に `.gitignore` に対象を追記し、`git commit` すれば以降は追跡されない。
