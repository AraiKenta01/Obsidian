## 主張

gitは `.git` フォルダの実体をリポジトリのルート以外の場所に分離できる。管理対象ディレクトリに `.git` という名前のテキストファイルを置き、実体の絶対パスを書くだけでよい。

## 展開

通常 `.git` フォルダはリポジトリルートに存在するが、これはデフォルトの挙動にすぎない。`gitdir:` という参照ファイルを使うことで実体を任意の場所に分離できる。主なユースケースはiCloudやDropboxなどのクラウド同期ツールと共存させる場合で、`.git` フォルダを同期対象外のディレクトリに逃がすことで[[複数の同期システムを同時使用してはいけない]]問題を回避できる。

## 具体例

管理対象リポジトリのルートに `.git` という名前のテキストファイルを作成し、以下を記述する。

```
gitdir: /Users/hoge/Documents/git-store/my-repo.git
```

これで `/Users/hoge/Documents/git-store/my-repo.git` に実体の `.git` フォルダが置かれ、リポジトリルートには参照ファイルだけが残る。
